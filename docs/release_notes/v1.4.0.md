# v1.4.0 更新内容

## Features

- 新增独立“模型”页面，可以直接选择聊天回复模型、测试连接并切换登录方式。
- 支持 Google OAuth 和本机 Gemini CLI 登录接入 Gemini，登录后可直接用于聊天回复。
- 支持 Qwen OAuth、Qwen Code 本机登录、DashScope API Key 和阿里云 Coding Plan API Key 接入，配置后可直接用于聊天回复。
- 支持复用本机 Codex / ChatGPT 登录状态接入 OpenAI，不需要一定再单独填写 API Key 才能开始聊天回复。
- 支持 Kimi Code 的 OAuth 和本机登录接入，配置后可直接用于聊天回复。
- 支持 Claude Code 本机凭据接入；当本机存在可复用凭据时，可直接用于聊天回复。
- 支持识别和导入豆包、元宝的网页登录状态，方便统一管理已有登录账号。
- 新增待审批回复，对新联系人、非白名单群、静音时段或敏感内容可先生成草稿、确认后再发送。
- 新增工作区备份与恢复，支持在改配置、换模型或排查问题前先做备份。
- 新增运行准备度检查和诊断快照导出，启动问题更容易定位。
- 新增命令行备份与恢复，以及离线评测命令，便于无界面环境下维护和发版前检查。

## Improvements

- 调整模型配置入口，模型相关设置从设置页拆分到独立“模型”页。
- 调整同一模型服务的登录管理方式，现在可以保留多种登录方式并在需要时切换备用方式。
- 调整恢复流程，恢复前会先做校验和预演，降低误覆盖当前数据的风险。
- 调整启动恢复逻辑，遇到权限不足、微信未启动或模型未配置时，会优先给出对应处理动作。
